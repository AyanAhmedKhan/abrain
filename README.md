# gbrain — Phase 0: Foundation

Implements the Phase 0 checklist from the build spec (§15): extensions,
all `gb_*` tables, all pgmq queues, the normalize worker with gate +
signal scoring, and the duplicate/idempotency verification test. Plus
two things the spec discussion added: the **sweeper** (self-healing
n8n→pgmq handoff) and the **dumb 3-node Whapi receiver** for n8n.

```
gbrain/
├── sql/
│   ├── 001_foundation.sql      # extensions, tables, indexes, RLS, status view
│   └── 002_queues.sql          # all 7 pgmq queues
├── workers/
│   ├── lib/
│   │   ├── db.py               # connection (direct/session pooler only)
│   │   ├── queues.py           # pgmq send/read/archive + DLQ helper
│   │   └── signal_score.py     # the LLM gate (§8)
│   ├── normalize.py            # Stage 1+2: envelope + gate
│   └── sweeper.py              # re-enqueues orphaned gb_raw rows
├── n8n/
│   └── whapi_receiver.json     # Webhook → (insert + enqueue, one CTE) → 200
├── tests/
│   └── test_dedup.py           # Phase 0 acceptance test
└── .env.example
```

## 1 · Database (5 min)

In the Supabase SQL Editor, run `sql/001_foundation.sql`, then
`sql/002_queues.sql`. Both are idempotent.

> **Resolved:** `gb_chunk.embedding` is `vector(768)`. Embeddings are
> generated by a batched embed worker calling **Gemini Embedding on
> Vertex AI** (MRL-truncated to 768 — best-in-class retrieval, ~free
> against Vertex credits); Supabase stores the vectors only. 768 also
> fits `text-embedding-005` (native, simpler) and Voyage `voyage-4`
> (if avoiding Google). `embed_model` is stored per chunk, so switching
> models later is selective, not a rebuild.

## 2 · Workers

```bash
cd gbrain
cp .env.example .env            # fill DATABASE_URL (direct, port 5432)
pip install "psycopg[binary]" python-dotenv --break-system-packages
set -a; source .env; set +a
python -m workers.normalize     # continuous; Ctrl-C to stop
```

Schedule the sweeper every 5 minutes (cron on the n8n host, or an n8n
Schedule → Execute Command node):

```bash
python -m workers.sweeper
```

## 3 · n8n receiver

Import `n8n/whapi_receiver.json`, attach your Supabase Postgres
credential (direct connection), activate, and point the Whapi webhook
(`mode: body`, events: `messages`) at the production URL. Set
`callback_persist: true` in Whapi so its retries cover n8n downtime.

The workflow is intentionally three nodes. The single CTE fans out the
`messages[]` array, inserts each with `ON CONFLICT DO NOTHING` (absorbs
Whapi retries), and enqueues in the same statement. If the query fails,
n8n returns non-200 → Whapi backs off and retries → the sweeper is the
second backstop. **Do not add logic here** — no Gemini calls, no Sheets,
no parsing. That was the old Notes Agent pattern; all intelligence now
lives behind the queue.

## 4 · Verify (the Phase 0 acceptance gate)

```bash
python -m tests.test_dedup
```

Proves: native-id dedup at the door, one envelope per logical item via
`idempotency_key`, replay-is-a-no-op, and the signal gate skipping junk
while queueing deal content. All five checks must pass before Phase 1.

## Observability from day one

```sql
select * from gb_pipeline_status;                                  -- what's stuck?
select queue_name, queue_length, oldest_msg_age_sec
from pgmq.metrics_all();                                            -- stage depths
select count(*) from gb_envelope where status='skipped';            -- skip rate numerator
select * from gb_dlq order by at desc limit 20;                     -- failures
```

Tune `SIGNAL_THRESHOLD` against the skip rate once real WhatsApp volume
flows: near-zero → too loose, very high → too tight.

## What Phase 1 adds (next)

Preprocess worker (text-layer/OCR/whisper.cpp + chunking), the extract
**batch submitter + result poller** with cached prompt prefix (Anthropic
Batch API), the **embed batch worker** (calls Gemini Embedding on Vertex,
writes vectors into pgvector), and cost logging — i.e., rerouting the
Dexter Venture Notes extraction through batch + cache. Keep the existing
Notes Agent running in parallel until the batched path produces
equivalent notes.
